# FULL PROJECT BRIEF (MVP)

> Legacy MVP brief from wallet-first planning phase.
> Current production flow is Monnify hosted checkout (not wallet-first).
> See `README.md` and `TODO.md` for current execution priorities.

## 1. Core Goals

- Allow users to buy social media accounts instantly using an internal wallet.
- Provide clean browsing of Platforms → Categories (Follower Ranges, age, type, most rated, etc.) → Purchase.
- Deliver purchased accounts ONLY inside the dashboard Purchases page.
- Allow users to become affiliates and earn commissions through unique codes.
- Provide admins full control over inventory, orders, wallet, affiliates, and text content.

## 2. User Flows

### Buyer Flow

1. Sign up/login (Email + Google OAuth).
2. Fund internal wallet.
3. Browse platforms and follower-range categories.
4. Choose quantity, apply affiliate discount.
5. Pay using wallet.
6. Receive accounts instantly in Purchases.
7. Access order history, wallet history, receipts.

### Affiliate Flow

1. User enables Affiliate mode in profile.
2. System generates code using INITIALS + running number.
3. Code shared manually.
4. Commissions tracked for every order using the code.

### Admin Flow

1. Manage Platforms, Categories, Products.
2. Import Batches and generate Accounts (Units).
3. Manage orders and deliveries.
4. Oversee wallets, affiliates, commissions.
5. View all site, business and sale analytics.
6. Edit UI microcopy. \*

## 3. Core Features

### Authentication

- Email/password, Google OAuth, password reset, email verification.

### Catalog Structure

- **Platforms** (TikTok, Instagram, etc.)
- **Categories** under platforms (Follower ranges): name, slug, min/max followers, price (₦), display flags.
- **Product** auto-synced per Category.

### Inventory (Batch-Based)

- Batch import: links list + credential blocks.
- Accounts paired automatically.
- Account states: available → assigned → delivered/failed.

### Wallet System

- Wallet funding via gateway (credits wallet).
- Purchases debit wallet only.
- Transactions: deposit, debit, refund.

### Checkout

- Totals = subtotal – discount.
- Discount line bold and visible.
- Order created → accounts allocated → delivered to dashboard.

### Delivery

- ONLY via Dashboard Purchases page.
- Email includes link back to Purchases.
- Mask/unmask, copy, resend.

### Affiliate System

- Code = INITIALS + running number.
- Checkout applies discount + logs commission.
- Affiliate dashboard with stats.

### User Dashboard

- Tabs: Purchases, Orders, Wallet, Affiliate, Profile.
- Copy editable through microcopy dictionary.

### Admin Panel

- Platforms & Categories CRUD.
- Batch import & inventory management.
- Orders overview & delivery control.
- Wallet oversight.
- Affiliate management.
- Basic analytics.
- Microcopy editor.

## 4. Technical Overview

- Hierarchical catalog (Platform → Category → Product → Batch → Units).
- Wallet-first economy.
- Affiliate codes auto-generated by initials + sequence.
- UI copy pull from dynamic dictionary.
- Allocation uses FIFO & safe locking.

## 5. Exclusions (Not in MVP)

- No WhatsApp/Telegram delivery.
- No direct payments for orders.
- No referral cookies.
