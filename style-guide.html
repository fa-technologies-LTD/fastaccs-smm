⬡ Product Themes

FA Technologies uses a two-tier colour architecture. All products share the master palette and dark-theme foundation above. Each product overrides the semantic --primary group with a product-specific accent. No product theme colour should appear outside its own product.

Rule: The FA Technologies master palette (Greens, Limes, Blues, Neutrals) is shared across all products and defined in the brand PDF. Product theme tokens are extensions — they override only --primary, --primary-strong, --glow-primary, and --focus. Everything else — backgrounds, typography, borders, spacing, radius, shadows — remains shared and unchanged.

⚠ Contrast check: #A67C00 on dark backgrounds can fall below WCAG AA for text smaller than 18px. Use #C9921A for any text, labels, or links. Reserve #A67C00 for button gradients and icons only.
/* ── FA Logistics: semantic override (fastaccs.com/logistics/style.css) ── */ :root { /* Gold raw tokens are already declared in the master :root above. */ /* Here we remap the shared semantic vars to the gold product theme. */ --primary: var(--fa-gold-600); /* #C9921A */ --primary-strong: var(--fa-gold-700); /* #A67C00 */ --glow-primary: 0 0 0 1px rgba(201,146,26,0.18), 0 10px 40px rgba(201,146,26,0.08); /* subtle — not gaming UI */ --focus: rgba(201,146,26,0.20); /* soft — premium feel */ } /* ── Gold usage rules ─────────────────────────────────────────────────── */ /* Use #C9921A (--primary) for: buttons, active tabs, price text, */ /* icons, small dividers, hover borders. */ /* Use #A67C00 (--primary-strong) for: button gradients, pressed state. */ /* Do NOT use gold for large fills or backgrounds — accent only. */ /* --fa-gold-400 (#D4A017) reserved for decorative thin lines only. */ /* ── FastAccs SMM: no override needed ─────────────────────────────────── */ /* --primary is already var(--fa-green-500) = #05D471 from master :root. */

9 Usage Guidelines

Best practices for applying the dark theme color system.

✓ Do
  • Use var(--primary) for CTAs and important actions
  • Use var(--text-muted) for secondary information
  • Apply gradients for depth on buttons and cards
  • Use opacity for text hierarchy (92%, 68%, 55%)
  • Maintain consistent border-radius across components
✗ Don't
  • Use pure white (#FFFFFF) for text (too harsh)
  • Mix light theme colors with dark theme
  • Use bright colors for large background areas
  • Exceed 3 levels of visual hierarchy
  • Use low contrast color combinations

Color Hierarchy

  1. Primary Actions: Use --primary (#05D471) for main CTAs, success states
  2. Interactive Elements: Use --link (#AAADFF) for links, secondary buttons
  3. Warnings/Alerts: Use --fa-lime-700 (#CADB2E) for attention-grabbing elements
  4. Text Hierarchy: --text (92%) → --text-muted (68%) → --text-dim (55%)

Accessibility

  • All text colors meet WCAG AA standards for contrast on dark backgrounds
  • Primary green (#05D471) has 7:1 contrast ratio on dark backgrounds
  • Focus states use 3px outlines with --focus color
  • Minimum touch target size: 44px × 44px (mobile)
  • Hover states provide clear visual feedback (180ms transition)
