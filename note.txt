FASTACCS EMAIL SYSTEM - FINAL REVIEW (CODE TRUTH)
Date: 2026-05-11
Status: Current as implemented

Scope
- This section lists ALL built-in email flows currently wired in code.
- It includes trigger timing, dedupe/cooldown behavior, and body/subject text patterns.

--------------------------------------------------
1) VERIFICATION CODE EMAIL
Type: verification
Trigger:
- User calls POST `/api/auth/verification/request` (initial request or resend).
- Sent by `ensureVerificationCode(...)`.

When:
- Immediate on request, if not rate-limited and resend cooldown allows.

Guards / Limits:
- Max 3 verification requests per hour per user.
- 60-second resend cooldown.
- OTP expires after `VERIFICATION_CODE_EXPIRY_MINUTES` (default 10 minutes).

Subject:
- `Your Fast Accounts verification code`

Body text:
- `Use this verification code to confirm your email: {code}`
- `This code expires in {minutes} minutes.`
- `Do not share this code with anyone.`

2) WELCOME EMAIL
Type: welcome
Trigger:
- After successful OTP verification via POST `/api/auth/verification/verify`.
- Sent by `sendWelcomeEmailIfNeeded(...)`.

When:
- Immediate after verification success.

Guards / Limits:
- Sent once per user (`welcome` + `status=sent` dedupe).

Subject:
- `Welcome to Fast Accounts`

Body text:
- `Welcome to Fast Accounts.`
- `You are all set to browse premium account tiers, complete secure checkout, and access purchases from your dashboard.`
- `Head to platforms to find the right accounts for your needs.`

CTA:
- `Browse Accounts` -> `{base_url}/platforms`

3) ORDER CONFIRMATION EMAIL
Type: order_confirmation
Trigger:
- Payment verified/marked paid (multiple paths call the same idempotent sender):
  - `/api/payments/verify`
  - `/api/webhooks/monnify`
  - reconciliation service
  - fulfillment service
- Sent by `sendOrderConfirmationEmailIfNeeded(orderId)`.

When:
- Immediate when payment is confirmed.

Guards / Limits:
- Transactional reservation + dedupe by `referenceId=order.id`.
- Prevents duplicate sends from concurrent/retry paths.

Subject:
- `Order confirmed — {FA-order-number}`

Body text:
- `Your payment was successful and your order has been confirmed.`
- `Order: {FA-order-number}`
- `Amount paid: ₦{total}`
- `Items:`
- `- {product} x{qty} ({line_total})`
- `Your account credentials are available on your dashboard. For security, we never send login details via email.`
